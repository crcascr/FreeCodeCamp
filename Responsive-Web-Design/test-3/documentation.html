Componentes

En React, toda la interfaz de usuario se construye a partir de componentes reutilizables. Un componente es una pieza encapsulada de código que combina la lógica y la interfaz de usuario en una sola unidad.

Los componentes pueden ser componentes de clase (class components) o componentes funcionales (functional components). La arquitectura basada en componentes promueve la reutilización de código, la modularidad y la facilidad de mantenimiento.

JSX

JSX es una extensión de sintaxis para JavaScript que permite escribir código similar a HTML dentro de archivos de JavaScript. Esto facilita la creación de componentes de React, ya que combina la estructura de la interfaz de usuario con la lógica de JavaScript en un solo lugar.

JSX se compila en llamadas a funciones de React que crean elementos virtuales del DOM, los cuales se actualizan de manera eficiente en el DOM real.

Ciclo de vida de los componentes

Los componentes de clase en React tienen métodos de ciclo de vida que se ejecutan en momentos específicos durante el proceso de renderizado y actualización. Estos métodos incluyen: componentDidMount Se ejecuta después de que el componente se monta en el DOM. componentDidUpdate Se ejecuta después de que se actualiza el componente. componentWillUnmount Se ejecuta antes de que se desmonte el componente y otros.

Estos métodos permiten realizar acciones adicionales, como obtener datos de una API, suscribirse a eventos o limpiar recursos cuando sea necesario.

Estados y props

Los componentes de React pueden tener un estado interno (state) y recibir datos de otros componentes a través de propiedades (props). El estado es un objeto JavaScript que representa los datos que pueden cambiar a lo largo del ciclo de vida del componente. Las props son datos inmutables que se pasan desde un componente padre a un componente hijo.

Cuando el estado o las props cambian, React actualiza de manera eficiente solo las partes de la interfaz de usuario que necesitan ser renderizadas nuevamente.

Eventos

En React, los eventos se manejan de manera similar a los eventos de JavaScript estándar, pero con una sintaxis ligeramente diferente. Los eventos se definen como funciones dentro de los componentes y se asignan a los elementos de la interfaz de usuario utilizando la sintaxis onClick onChange onSubmit, etc.

React utiliza un sistema de eventos sintéticos que garantiza un comportamiento consistente en diferentes navegadores.

Renderizado condicional

El renderizado condicional permite mostrar u ocultar partes de la interfaz de usuario según ciertas condiciones. Esto se puede lograr utilizando declaraciones condicionales (if, else), operadores ternarios u operadores lógicos en el JSX.

Además, React proporciona técnicas como el renderizado de listas y el renderizado de componentes anidados, lo que permite crear interfaces de usuario dinámicas y adaptables.

Hooks

Los Hooks son funciones especiales que permiten usar características de React (como el estado y el ciclo de vida) en componentes funcionales. Algunos de los Hooks más comunes son:

  • useState: Para manejar el estado
  • useEffect: Para reemplazar los métodos de ciclo de vida
  • useContext: Para acceder al contexto de React
  • useReducer: Para manejar estados complejos.

Los Hooks promueven la reutilización de lógica y facilitan la creación de componentes funcionales más complejos.

React Router

React Router es una biblioteca de enrutamiento que permite manejar las rutas en una aplicación React de manera declarativa. Permite definir rutas y asociarlas con componentes específicos, lo que facilita la creación de aplicaciones de una sola página (SPA) con una navegación fluida.

React Router proporciona componentes como:

  • Router: Para configurar y manejar la navegación.
  • Link: Para crear enlaces de navegación.
  • Switch: Para controlar la navegación.
  • Route: Para renderizar los componentes correspondientes.

Para controlar la navegación y renderizar los componentes correspondientes según la URL actual.

Redux

Redux y la Context API son soluciones para el manejo del estado global en aplicaciones complejas de React. Redux es una biblioteca externa que implementa un patrón de arquitectura llamado Flux, lo que facilita la gestión del estado de manera predecible y centralizada.

La Context API, por otro lado, es una característica incorporada en React que permite compartir datos entre componentes sin tener que pasar props manualmente a través de la jerarquía de componentes.

Renderizado en el servidor

El renderizado en el servidor (SSR) es una técnica que permite renderizar los componentes de React en el servidor y enviar el HTML renderizado al cliente.

Esto mejora el rendimiento y la experiencia del usuario, ya que el contenido se muestra más rápidamente y los motores de búsqueda pueden indexar mejor el contenido. El SSR puede lograrse mediante frameworks como Next.js o soluciones personalizadas que utilizan Node.js en el servidor.
